利用 jwks key 转化为公钥，并进行验签：
`n`：Base64 编码的 modulus（模数 N）
`e`: Base64 编码的 exponent（指数 e，通常是 65537）


步骤：

① 使用 模数`n`、指数`e` 生成公钥：

### Hutool 工具类
	PublicKey publicKey = KeyUtil.getRSAPublicKey(new BigInteger(1, Base64.decode(n)), new BigInteger(1, Base64.decode(e)));

### 原生 JAVA
	KeyFactory factory = KeyFactory.getInstance("RSA");
	RSAPublicKeySpec publicKeySpec = new RSAPublicKeySpec(new BigInteger(1, java.util.Base64.getUrlDecoder().decode(n)), new BigInteger(1, java.util.Base64.getUrlDecoder().decode(e)));
	PublicKey publicKey = factory.generatePublic(publicKeySpec);

### 注释：`n` 是一个 2048/4096 位的正整数，必须使用 BigInteger modulus = new BigInteger(1, nBytes); 强制转为正数。

### 原生 JAVA
    String[] parts = token.split("\\.");
    String header = parts[0];
    String payload = parts[1];
    String signature = parts[2];

    // 构造待验签数据：header.payload
    String signInput = header + "." + payload;
    byte[] data = signInput.getBytes(java.nio.charset.StandardCharsets.UTF_8);

    // 解码 Base64Url 签名（注意：JWT 使用 URL 安全编码）
    byte[] signatureBytes = java.util.Base64.getUrlDecoder().decode(signature);

    // 初始化 SHA256withRSA 验签器
    Signature verifier = Signature.getInstance("SHA256withRSA");
    verifier.initVerify(publicKey);
    verifier.update(data);
    System.out.println(verifier.verify(signatureBytes));

  关于 Base64Url：
    `+` 在 URL 查询参数中会被解码为空格（application/x-www-form-urlencoded）,`/` 是路径分隔符，`=` 是键值对赋值符号，在某些上下文中需转义。Base64Url 编码时进行如下转换，去掉填充字符 `=`，将 `+` 替换为 `-`，将 `/` 替换为 `_` 。简单处理：n = n.replace('=', '').replace('+', '-').replace('/', '_')。Java 8+ 提供了 Base64.getUrlDecoder()，原生支持 Base64Url 解码，自动将 `-` 替换为 `+`，`_` 替换为 `/`，自动补足缺失的 `=` 填充。简单处理：n = n.replace('-', '+').replace('_', '/');
